GASCII visual & interaction spec

Retro-Mac-inspired, flat and crisp. Companion to mockups and workflows. Everything below is stated so a coding agent can implement it directly.

1 · Design principles

  • Flat, 1px, square. No gradients, no rounded corners above 0px, no soft shadows. Depth comes only from hard offset shadows (2–4px, no blur) on floating surfaces.
  • Inversion = selection. The selected state of any control (tool, tab, swatch, segment) inverts fg/bg. No accent-colored fills for selection in chrome.
  • Accent is reserved for the canvas. The blue accent appears only on canvas overlays: selection marquee, cell cursor, size tags. Chrome stays monochrome.
  • Monospace = data. Fragment Mono for anything that is content or measurement (glyphs, coordinates, sizes, hex values, hints). Instrument Sans for labels and controls.
  • The document is the star. Chrome is quiet warm gray; the canvas sits on a slightly darker desk with the doc as a bordered card.

3 · Typography

  • UI face: Instrument Sans (ship the TTF; fallback: any humanist sans). Sizes: 13px menus/controls, 12px buttons/segments/labels-strong, 11px field labels, 600 weight for titles only.
  • Mono face: Fragment Mono (or the canvas font itself). Sizes: 13px palette glyphs, 11px status bar, 10px section micro-labels (letter-spacing +0.08em, uppercase), 8px L/R badges.
  • Canvas font: a bundled monospace with full box-drawing + block-element coverage. Cell metrics derive from the font: cell w = advance width, cell h = line height at the chosen zoom.
  • No text under 10px anywhere except the 8px L/R tool badges.

4 · Layout & spacing

  • Spacing scale: 4 / 8 / 12 / 16 / 24. Panel padding 12px. Control gaps 8px; group gaps 14px.
  • Row heights: title bar 30, menu bar 26, options bar 40, status bar 28. Sidebar width 208 fixed.
  • Structural borders (window edge, sidebar edge, options/status bar separators) use border.strong; intra-panel separators use border.soft.
  • Toolbox: 3-column grid, 42px cells, 1px gaps filled by the grid's border color (cells butt together, MacPaint-style).
  • Palette swatches 26×26, 3px gaps, 6 per row. Recent row identical sizing.
  • The document floats centered in the canvas area on the desk color, with a strong 1px border and a 3px hard offset shadow. Minimum 28px desk margin around it.

5 · Component rules

Segmented control — 1px strong border around the group; 1px strong dividers; selected segment inverts (bg.inverse / fg.inverse); 4–6px vertical, 10–12px horizontal padding. Used for: L/R binding, brush shape, tabs, presets, export format, zoom cluster (borders soft for status bar).
Buttons — 1px strong border, transparent fill, 6px×16px padding. Default/primary button: inverted fill + 2px hard offset shadow. No hover fills; hover = underline on menu items, 1px border darken elsewhere.
Steppers — bordered group: [−][value][+]; value cell is mono, min 30px wide. Scroll-wheel over the value adjusts it.
Checkboxes — 12px square, 1px strong border; checked = inverted with ✓.
Tool cell — 42px, icon 17px at 1.4px stroke. States: idle (panel bg), hover (border.soft fill), active-L (inverted + mono “L” badge top-right), bound-R (mono “R” badge top-right, secondary color). Tooltip: name + shortcut.
Glyph swatch — 26px, mono 13px glyph. Idle: soft border. Hover: strong border. Selected: inverted. Clicking a swatch also pushes it into RECENT (front, dedup, cap 6, persisted per app).
Color wells — 24px squares, FG overlaps BG by 4px (FG top-left). Click opens picker (full RGB, per-cell fg+bg). Swap control ⇄ and shortcut X.
Dialogs — modal, centered, 340–360px wide; title bar 28px with pinstripe fills and a 12px close box; 16px body padding; buttons right-aligned, primary last.
Canvas overlays — cell cursor: 1px accent outline on hovered cell. Marquee: 1px dashed accent + 8% accent fill + size tag (accent bg, canvas-color text, mono 10px) top-right, outside. Grid: off by default; optional 1px lines at 4% fg opacity (View ▸ Grid).

6 · Interaction spec

  • L/R binding: click or shortcut sets L; right-click a tool cell sets R. Options bar shows one binding at a time via its L/R segment; using a mouse button on canvas auto-selects that segment. Both bindings persist across sessions.
  • Shortcuts: P Pencil, E Eraser, I Eyedropper, T Text, F Fill, R Rectangle, L Line, S Selection, B Brush, X swap FG/BG, Ctrl+Z/Y undo/redo, Ctrl+C copy selection, Ctrl+Shift+C copy all as text, Ctrl+S save, Ctrl+Shift+E export, +/− zoom, Ctrl+0 fit.
  • Selection lifecycle: see workflow W2 — float on drag, Alt-drag copies, arrows nudge, Enter/click-out commits, Esc cancels.
  • Eyedropper samples glyph + FG + BG of the cell; respects WRITE toggles when re-applying.
  • Zoom: steps 50/100/150/200/300/400%; Fit computes from viewport. Ctrl+wheel zooms about the cursor cell.
  • Theme: View ▸ Theme ▸ Light/Dark/System. Chrome tokens swap; document surface does not.
  • Menus: File: New… / Open… / Save / Save As… / Export… / Recent Files. Edit: Undo / Redo / Copy Selection / Copy All as Text / Paste / Resize Canvas…. View: Zoom In/Out/Fit / Grid / Theme.

7 · egui implementation notes

  • Set global Visuals: widgets.*.rounding = 0, expansion = 0, window_shadow/popup_shadow → hard offset (offset (3,3), blur 0), stroke widths 1.0.
  • Map tokens to Visuals: panel_fill = bg.panel, window_fill = bg.chrome, faint_bg_color = border.soft, override selection/hyperlink colors to the monochrome scheme; build two Visuals structs (light/dark) and swap.
  • Inverted-selection widgets (tool cells, swatches, segments) are custom: allocate with ui.allocate_exact_size + paint rect/border/text manually rather than styling SelectableLabel.
  • Register fonts via FontDefinitions: Instrument Sans → Proportional, Fragment Mono (or canvas font) → Monospace. Set text styles to the sizes in §3.
  • Layout: TopBottomPanel (title/menu/options/status) + SidePanel::left(208.0).resizable(false) + CentralPanel with desk fill; paint the document + overlays on a Painter layer.
  • Pinstripes: painter loop of 1px horizontal lines every 3px at the opacities in the mockup (0.28 light / 0.18 dark) — skip if cheaper to omit; they're decorative only.
  • Persist: theme, L/R bindings + per-tool options, RECENT glyphs, last export settings (e.g. via eframe::Storage).
